# セッション引き継ぎ (handoff)

次のセッションの Claude が、この会話を読まずに作業を再開できる状態を作る。

## 手順

1. `git status` と `git log --oneline -5` で今セッションの変更を把握する
2. `docs/STATUS.md` を更新する:
   - 「現在地」: フェーズと作業中マイルストーンを現状に合わせる
   - 「次にやること」: 具体的な次の 1〜3 タスク。作業が中途半端なら「どこまで終わってどこから再開するか」をファイル名付きで書く
   - 「申し送り・注意点」: 今セッションで判明したハマりどころ・保留した判断・ユーザーからの指示を追記(古くなった項目は削除してよい)
   - 「セッション履歴」に 1 行追加
3. マイルストーンが完了した場合は `docs/roadmap.md` のチェック(⬜→✅)を更新する
4. 設計・技術選定を変更した場合、`docs/decisions.md` と `docs/architecture.md` に反映済みか確認する
5. 未コミットの変更をすべてコミットする(Conventional Commits)。作業途中なら `wip:` プレフィックスで構わない
6. 最後に、ユーザーへ「完了したこと / 次回やること / ユーザー側に必要なアクション(あれば)」を短く報告する

## 注意

- STATUS.md は「次の Claude への手紙」。会話の文脈を知らない読者に伝わる書き方をする(「さっきの件」「上記の通り」は禁止)
- コミットせずにセッションを終えない
