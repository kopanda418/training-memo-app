# 開発計画(ロードマップ)

マイルストーンは 1 つずつ順に完了させる。各マイルストーンは「1〜2 セッションで完了できる粒度」を意図している。完了時は STATUS.md を更新し、このファイルのチェックを付ける。

## Ver 1.0 までのマイルストーン

### M0: プロジェクト基盤 ✅ (2026-07-03)

- Vite + React + TypeScript (strict) + Tailwind の雛形
- ESLint / Prettier / Vitest 設定、`npm run dev|build|test|lint` が通る
- vite-plugin-pwa 導入(manifest・アイコン仮・オフライン動作)
- GitHub リポジトリ作成、GitHub Actions で Pages へ自動デプロイ
- 下部タブ 4 画面(中身は空)+ hash ルーティング
- **完了条件: iPhone のホーム画面に追加して起動でき、機内モードでも開ける**

### M1: データ層 ✅ (2026-07-03)

- Dexie スキーマ v1(architecture.md の全テーブル)
- repository 関数(セット追加/更新/削除、日の取得、種目×タグの履歴取得)+ ユニットテスト
- JSON エクスポート/インポート(全テーブル、バージョン番号付き)
- デフォルト種目マスタの投入(部位別の代表種目)
- **完了条件: テストが通り、エクスポート→全削除→インポートで復元できる**

### M2: 記録入力画面(コア UX)⬜

- 日付ヘッダ(前後日移動・カレンダーへ)
- 種目選択モーダル(部位タブ → 種目 → タグ選択/新規タグ)
- セット行の入力(重量・回数ステッパー、補助チェック、前回値プリフィル、自動保存)
- 「前回コピー」(同一種目×タグの直近記録を一括コピー)
- 場所の記録(過去候補から選択 + 新規テキスト)
- **完了条件: 実機でジム 1 回分の記録が引っかかりなく入力できる(最重要マイルストーン。ここで UX を納得いくまで調整)**

### M3: 履歴・コピー/移動 ✅ (2026-07-03)

- カレンダー表示(記録日マーク → 日タップで記録画面へ)
- 種目別履歴(タグフィルタチップ)
- 日単位・種目単位のコピー/移動(トランザクション、移動後の空の日の掃除)
- **完了条件: 過去日の記録を別日へコピー/移動し、履歴・当日画面に正しく反映される**

### M4: 分析(週間ビュー・グラフ・MAX)⬜

ユーザーの主目的(週間量の漸進確認)に直結する順で実装する(2026-07-05 に再定義)。

- M4a: ウォームアップフラグ(セット行の W トグル、薄色表示、コピー/テンプレート引き継ぎ)
- M4b: 週間ビュー(グラフタブ初期表示): 今週サマリ+基準週(‹›で変更)との差分、種目別内訳(タグフィルタ付き)、週次推移バー 8〜12 週。集計ルールは requirements.md の「分析」参照
- M4c: 種目×タグ選択 + 指標 4 種(最大重量/推定1RM/総負荷量/セット数)の折れ線グラフ
- M4d: MAX 記録一覧(種目×タグ、ウォームアップ除外)+ 記録入力時の MAX 更新トースト
- **完了条件: 週間ビューで先週比の差分と種目別不足分が確認でき、タグ違いの同一種目が別系列でグラフ表示され、MAX 更新がタグ単位で通知される**

### M5: インターバルタイマー + Wake Lock ⬜

- プリセット/任意秒のカウントダウン、終了音(Web Audio)、どの画面からも起動
- タイマー動作中の Screen Wake Lock(設定で ON/OFF、visibilitychange 再取得)
- **完了条件: 実機でタイマー中に画面が消灯せず、終了時に音が鳴る**

### M6: テーマ・設定・仕上げ ⬜

- ライト/ダーク/システム追従(theme-color 連動)
- マスタ管理画面(種目・タグ・場所の編集/アーカイブ)
- `navigator.storage.persist()` 要求、バックアップ推奨導線
- アイコン・スプラッシュ等の本番アセット、全体の動作確認
- **完了条件: Ver 1.0 タグを打ち、日常利用を開始できる**
