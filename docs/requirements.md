# 要件定義

## 目的

iPhone で使う筋トレ記録 PWA。「筋トレmemo」(https://kintorememo.jp/) の記録体験をベースに、下記の独自要件を加える。個人利用・費用ゼロ。

## 機能要件

### 1. トレーニング記録(コア機能)

- 日単位で記録する。1 日 = 場所 + 複数の「種目×タグ」ブロック + 各ブロック内の複数セット
- セットの記録項目: 重量(kg / lbs)、回数(レップ数)、補助の有無、メモ(任意)
- 種目は部位(胸・背中・脚・肩・腕・腹・有酸素など)別のマスタから選択。カスタム種目の追加・編集・アーカイブが可能
- **種目×タグ(差分要件)**: 種目選択後に任意でタグ(例: 高重量日・中重量日・低重量日)を 1 つ付与できる。同じ種目でもタグが異なれば履歴・グラフ・MAX 判定上は別種目として扱う。タグなしも 1 つの区分として扱う
- 前回記録のコピー: 同じ種目×タグの直近記録をワンタップで当日にコピーして編集開始できる
- **日付間コピー/移動(差分要件)**: 任意の日の記録全体(または種目単位)を、任意の日へコピーまたは移動できる
- **トレーニング場所(差分要件)**: 日ごとに場所を記録できる。過去に入力した場所が選択肢として出て、新規はテキスト入力で追加

### 2. 履歴

- カレンダー表示: 記録がある日にマークを付け、日付タップでその日の記録へ
- 種目別履歴: 種目を選ぶと日付降順で全セット履歴を表示。**タグでフィルタ可能(差分要件)**

### 3. 分析

**週間ビュー(2026-07-05 追加。グラフタブの初期表示)** — 週単位の量の漸進をノイズなしで確認する

- 週は月曜始まり(将来は設定で変更可能な構造にする)
- 今週のサマリ: 総セット数・総ボリュームロード(実効負荷×実績レップの合計)と、基準週との差分(増=緑/減=赤)
- 基準週: デフォルトは先週。‹ › で 2 週前・3 週前…に変更できる(先週がイレギュラーな場合への対応)
- 種目別内訳: **種目単位**で「今週 / 基準週 / 差分」のセット数・ボリュームを表示。マイナス = 今週の不足分。**タグでフィルタ可能**
- 週次推移: 過去 8〜12 週のセット数 / ボリュームの棒グラフ(トレンド俯瞰)
- **集計ルール(ノイズ除去)**: ウォームアップセットと実績レップ空欄のセットは、セット数・ボリュームとも集計から除外する
- **ウォームアップフラグ**: セットに専用トグル(W)を新設。属性(自由テキスト)とは別の第一級フラグとし、コピー・テンプレートでも引き継ぐ。ウォームアップ行は薄く表示
- 自重セットのボリューム: 体重登録済みなら「体重+加重」で計算。体重未登録の自重セットは集計から除外し、除外した旨を注記表示

**折れ線グラフ・MAX 記録(従来どおり)**

- 折れ線グラフ: 指標は「最大重量」「推定 1RM」「総負荷量(重量×回数の合計)」「セット数」。表示対象は種目×タグで絞り込む
- MAX 記録: 種目×タグ単位で最大重量・最大回数・推定 1RM のベストを保持(ウォームアップセットは判定から除外)。**記録入力時にベスト更新を検知したら画面内で祝福通知を表示(差分要件: 種目×タグ単位)**
- 推定 1RM の自動計算(Epley 式: `weight × (1 + reps / 30)` を採用。式は設定で将来変更可能な構造にする)

### 4. インターバルタイマー

- セット間の休憩タイマー。プリセット秒数(例 60/90/120/180 秒)+ 任意秒数
- 終了時は音(Web Audio)+ 画面表示で通知(iOS PWA はロック中のプッシュ通知・バイブが使えないため)
- **Wake Lock(差分要件)**: タイマー動作中は Screen Wake Lock API で画面ロックを抑止する。設定で ON/OFF 可能。タブ非表示で解除されたら復帰時に再取得

### 5. テーマ・設定

- **ライト/ダークモード切替(差分要件)**: ライト固定・ダーク固定・システム追従の 3 択
- 重量単位のデフォルト(kg / lbs)
- データのエクスポート/インポート(JSON 1 ファイル。全テーブルを含む。バックアップと機種変更用)
- マスタ管理: 種目・タグ・場所の追加/名前変更/アーカイブ

### 除外事項(作らない)

- 体重・体脂肪などのボディ管理画面
- アカウント・クラウド同期・SNS 連携(Ver1.0 では除外。バックログ参照)
- プッシュ通知(iOS PWA の制約が大きく、タイマーは Wake Lock + 音で代替)
- 広告・課金

## 非機能要件

| 項目       | 要件                                                                                                                    |
| ---------- | ----------------------------------------------------------------------------------------------------------------------- |
| 対応環境   | iPhone Safari(iOS 16.4 以上)。ホーム画面追加(standalone)での利用が前提                                                  |
| オフライン | 初回ロード後は全機能がオフラインで動作                                                                                  |
| 応答速度   | 記録入力の操作→画面反映は体感遅延なし(目安 100ms 未満)。入力経路にネットワークを挟まない                                |
| 費用       | ゼロ。静的ホスティング(GitHub Pages)のみ、バックエンドなし                                                              |
| データ保全 | IndexedDB + `navigator.storage.persist()` を要求。JSON エクスポートで手動バックアップ可能。エクスポート推奨を UI で促す |
| コード管理 | GitHub リポジトリ。main への push で自動デプロイ                                                                        |

## データ量の想定

週 3〜4 回 × 1 回 20〜30 セットで年間約 5,000 セット。10 年分でも数万レコードであり、IndexedDB とクライアント集計で十分な規模。
