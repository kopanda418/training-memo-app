# フィードバック対応 第7弾 / v1.0.1 改善計画(2026-07-06)

Ver 1.0.0 リリース後の使い勝手改善。ユーザー意思決定済み。実装は着手指示待ち。

## 確定した意思決定

1. **セット属性は「1 つ」→「複数」へ変更**(2026-07-05 の決定を上書き)
2. **セット追加時、属性も前セットからコピー**する(重量・単位・自重フラグと同様)
3. **目標レップ入力を廃止し、その欄を RPE(数値)入力にする**
   - `sets.rpe?: number`(小数可: 8.5 など)を新設。`targetReps` は UI から撤去(既存データは保持するが表示しない)
   - 新規セット追加時、RPE も前セットからコピーする(ユーザー決定 2026-07-06。属性・重量等と揃える)
4. **移行データの再生成**: 旧アプリのメモ欄を解析して反映
   - 属性へ変換: 「左」「右」「フル」「QL」
   - RPE(数値)へ変換: 「RPE8」「RPE 8.5」等の RPE+数値 表記
   - 変換ルール: メモをトークン分割(スペース・読点・カンマ等)し完全一致のみ属性化。RPE は正規表現で数値抽出。変換に使った部分はメモから除去し、残りの自由文はメモとして保持
   - 実装時に実データのメモ一覧を確認してルールを最終調整する
