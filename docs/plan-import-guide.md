# 筋トレメモアプリ 連携ガイド(トレーニングプラン→予定インポート用)

別プロジェクト(週次プラン生成側)が「筋トレメモアプリ」にプランを取り込む import 用 JSON を
組み立てるための最小限の情報。アプリ本体のソースは別リポジトリ(非公開)にあり、この文書だけで完結する。

## 連携経路: プラン取り込み(追加専用、実装済み)

設定画面の「プラン取り込み(追加のみ)」から、専用フォーマットの小さな JSON を取り込める
(`docs/decisions.md` ADR-010)。**既存データは一切書き換えない。** 内部 id は不要で、
種目名・タグ名だけでプランを組み立てられる。

- まだ記録がない `date + 種目 + タグ` の組み合わせにだけセットを追加する
- 既に記録がある組み合わせは**丸ごとスキップ**(冪等。同じファイルを何度取り込んでも重複しない)
- 未登録の種目・タグは name 一致で新規作成する(種目は `bodyPart` 指定が必須)
- 取り込み前にプレビュー(追加/スキップ/新規作成の一覧)が出るので、name の表記ゆれによる
  意図しない新規種目作成はその場で確認できる

旧来の「全データエクスポート→編集→復元(全置換)」という往復は不要になった。

## JSON フォーマット

```ts
interface PlanImportFile {
  app: 'training-memo-app'
  kind: 'plan-import' // 通常バックアップと区別する識別子(固定)
  formatVersion: 1
  days: {
    date: string // "YYYY-MM-DD"
    location?: string // 場所名(任意)。新規に日を作る場合のみ適用(既存日は上書きしない)
    items: {
      exercise: string // 種目名。既存の種目マスタと name 完全一致で照合
      bodyPart?: string // 新規種目を作る場合のみ必須(例: "胸","背中","脚","肩","腕","腹")
      tag?: string // タグ名(省略でタグなし)。未登録なら新規作成
      sets: {
        weight: number // 目標重量(必須)
        unit?: 'kg' | 'lbs' // 省略時はアプリ設定の defaultUnit
        reps?: number // 省略時 0 = 「未実施(予定)」。週間集計・MAX判定から自動除外される
        isBodyweight?: boolean
        memo?: string // 目標メモ(RPEなど自由記述)
      }[]
    }[]
  }[]
}
```

### 例

```json
{
  "app": "training-memo-app",
  "kind": "plan-import",
  "formatVersion": 1,
  "days": [
    {
      "date": "2026-07-14",
      "items": [
        {
          "exercise": "ベンチプレス",
          "tag": "高重量日",
          "sets": [
            { "weight": 82.5, "memo": "目標 5reps RPE8" },
            { "weight": 82.5, "memo": "目標 5reps RPE8" }
          ]
        }
      ]
    }
  ]
}
```

## 生成側で気をつけること

- **種目名・タグ名はこのアプリの既存マスタと完全一致(trim後)させる。** 表記ゆれ(例:
  "スクワット" vs "バーベルスクワット")は別種目として新規作成されてしまう。可能なら
  ユーザーに既存の種目・タグ名一覧を一度確認してもらい、生成側でも固定の名称リストとして
  保持するのが安全
- 新規種目を作る場合は `bodyPart` を必ず入れる(省略するとその項目はエラーになり取り込まれない)
- `reps` は基本的に省略してよい(0 = 予定。ジムで実施後にユーザーがアプリ上で実績へ書き換える)
- **同じ日付×種目×タグへ内容を差し替えたい場合は非対応。** スキップされるだけなので、
  変更したい場合はユーザーがアプリ側で既存セットを編集/削除してから再取り込みする必要がある
- ファイルは小さくてよい(直近1〜2週間分の新規プランだけを含めれば十分。過去分・実績データは
  一切含める必要がない)
